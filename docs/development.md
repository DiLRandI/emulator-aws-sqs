# Development

## Local workflow

Build the binary:

```bash
make build
```

Run in the foreground:

```bash
make run
```

Run in the background:

```bash
make dev-up
make dev-down
```

Background server state:

- PID file: `.dev/sqsd.pid`
- log file: `.dev/sqsd.log`

## Test workflow

Lightweight in-process Go tests:

```bash
make test-unit
```

Go integration tests that spawn the real `sqsd` binary:

```bash
make test-integration
```

AWS CLI integration tests:

```bash
make test-cli
```

Full local validation:

```bash
make ci
```

Focused test examples:

```bash
go test ./internal/tests -run TestRawJSONLifecycleSigned -v
go test ./internal/tests -run TestSDKFIFOAndDLQResolverV2 -v
```

## Debugging tips

- Run `make dev-up` and watch `.dev/sqsd.log`.
- Use `SQS_LOG_LEVEL=DEBUG make run` for more verbose server logs.
- Run a single raw protocol test with `go test ./internal/tests -run TestRaw -v`.
- Reproduce CLI behavior directly with [tests/aws_cli_integration.sh](/home/deleema/learning/emulator-aws-sqs/tests/aws_cli_integration.sh).
- Reproduce the host-to-container AWS CLI path with [tests/aws_cli_docker_smoke.sh](/home/deleema/learning/emulator-aws-sqs/tests/aws_cli_docker_smoke.sh).
- If local state looks wrong, remove `sqs.db`, `sqs.db-shm`, and `sqs.db-wal`, or run `make clean`.
- If Docker state looks wrong, run `make docker-clean`.
- `make docker-run` and `make docker-compose-up` share the same named Docker volume by default.

## Environment variables

The most useful runtime variables are:

- `SQS_LISTEN_ADDR`
- `SQS_PUBLIC_BASE_URL`
- `SQS_REGION`
- `SQS_ALLOWED_REGIONS`
- `SQS_ACCOUNT_ID`
- `SQS_SQLITE_PATH`
- `SQS_SQLITE_DSN`
- `SQS_AUTH_MODE`
- `SQS_CREDENTIALS_FILE`
- `SQS_DEFAULT_ACCESS_KEY_ID`
- `SQS_DEFAULT_SECRET_ACCESS_KEY`
- `SQS_DEFAULT_SESSION_TOKEN`
- `SQS_LOG_LEVEL`
- `SQS_CREATE_PROPAGATION`
- `SQS_DELETE_COOLDOWN`
- `SQS_PURGE_COOLDOWN`
- `SQS_ATTRIBUTE_PROPAGATION`
- `SQS_RETENTION_PROPAGATION`
- `SQS_LONG_POLL_WAKE_FREQUENCY`

The Makefile also exposes workflow variables with defaults:

- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `AWS_SESSION_TOKEN`
- `AWS_REGION`
- `SQS_HOST`
- `SQS_PORT`
- `SQS_ENDPOINT`
- `SQS_DB_PATH`
- `SQS_SQLITE_DSN`
- `TEST_SQS_HOST`
- `TEST_SQS_PORT`
- `TEST_SQS_ENDPOINT`
- `TEST_SQS_DB_PATH`
- `DOCKER_IMAGE`
- `DOCKER_CONTAINER`
- `DOCKER_VOLUME`
- `DOCKER_HOST_PORT`
- `DOCKER_ENDPOINT`
- `DOCKER_PUBLIC_BASE_URL`
- `DOCKER_SQLITE_PATH`

Example:

```bash
AWS_REGION=us-west-2 SQS_PORT=9444 SQS_DB_PATH=local-dev.db make dev-up
TEST_SQS_PORT=19444 TEST_SQS_DB_PATH=.tmp/cli-tests.db make test-cli
DOCKER_HOST_PORT=9444 DOCKER_PUBLIC_BASE_URL=http://127.0.0.1:9444 make docker-run
```

## Code generation

The SQS model registry is generated by [tools/gensqs/main.go](/home/deleema/learning/emulator-aws-sqs/tools/gensqs/main.go).

Regenerate it with:

```bash
go run ./tools/gensqs \
  -service-model /path/to/service-2.json.gz \
  -paginators /path/to/paginators-1.json \
  -out ./internal/model/registry_gen.go
```
